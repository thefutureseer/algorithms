Input Validation / Finite-State Machines
Why it matters: For parsing requests, managing secure login flows, or rate limiting.

Algorithms:

Finite state machines (FSMs) for input parsing

Automata to recognize valid patterns

Counters and token buckets (rate limiting)

Example Interview Questions:

Build an FSM to validate multi-step login input (e.g., username → password → OTP).

Implement a rate limiter that allows N requests per minute per user.
